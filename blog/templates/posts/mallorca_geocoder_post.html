Actualmente las aplicaciones basan mucha de su funcionalidad en la información geográfica. Las redes sociales: Facebook, Twitter, Instagram, ... utilizan información de este tipo para geoposicionar a sus usuarios.
Disponer de grandes cantidades de este tipo de información puede resultar de mucha utilidad para la realización de estudios estadísticos con el objetivo de obtener información útil sobre los usuarios.

Esta información geográfica suele representarse en mapas. Para ello contamos con librerías como Google maps o OpenStreetMaps. En general estas librerías, dado el nombre de una zona (país, ciudad, municipio, ...) te proporcionan su ubicación. Este proceso se conoce como geocoding. En mi caso, el problema que se me planteó fue el contrario: dado un punto geográfico necesitaba su información. Este proceso se conoce como reverse geocoding.

Dado un trazado GPS, es decir, una ruta por la isla de Mallorca; necesitaba saber qué municipios de la isla recorría. El problema lo podemos reducir a, dado un punto geográfico, obtener el municipio al que pertenece.
El geocoder de Google ya dispone de esta funcionalidad y con este simple código en Javascript ya podemos obtener el municipio de un punto.

Pero había un problema. Las rutas GPS disponen, cada una, de una cantidad de puntos muy elevada y Google limita el número de peticiones. Por esta razón decidí crear mi propio geocoder.

La idea era realizar un geocoder que no requiriera internet para obtener el municipio de un punto. En la web del Centro Nacional de Información Geográfica disponen de archivos que definen las líneas fronterizas de los municipios de todas las províncias de España. Así que descargué las líneas municipales de Mallorca y las transformé a un formato más común: GPX. Con los polígonos municipales definidos lo único que tuve que hacer fue un programa que comprobara la pertenencia de un punto a cada uno de los polígonos.

El algoritmo, para comprobar la pertenencia de un punto a un polígono, se basa en el trazado de rayos. Consiste en trazar una recta y contar el número de intersecciones que se producen con el polígono. Si el número de intersecciones es impar quiere decir que el punto se encuentra en el interior del polígono. En contraposición, si el número de intersecciones es par, el punto se encuentra fuera del polígono.

En el repositorio de Github tenéis el código completo junto con los archivos GPX que definen las fronteras de cada municipio de Mallorca.
